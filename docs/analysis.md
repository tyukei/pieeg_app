# 脳波の分析・集計のしくみ

生の脳波(16ch)から、画面に出る「バンドパワー」「集中/リラックス」「頭部マップ」までの流れを、
順を追って説明します。実装は `acquirer/codec.py`・`server/aggregate.py`・`web/src/fft.ts`・
`web/src/mind.ts`・`web/src/topography.ts` にあります。

---

## 全体像

```
生波形(μV) ──窓で切り出し──▶ 前処理 ──FFT──▶ 周波数ごとの強さ(PSD)
                                                    │
                        ┌───────────────────────────┤
                        ▼                            ▼
                 バンドパワー(δθαβγ)          集中度 / リラックス度
                        │                            │
                        ▼                            ▼
                 頭部トポグラフィ           認知状態トラジェクトリ(3D)
```

---

## 1. 生波形をつくる（24bit → μV）

PiEEG-16 は 2 個の ADS1299 チップで **16ch を 250Hz、24bit** で読み取ります。24bit の生の数値は
そのままでは電圧ではないので、マイクロボルト(μV)に変換します。

- フルスケール ±4.5V を 24bit(符号付き)で表現し、増幅率(PGA gain = 6)で割る
- 変換係数: `1,000,000 × 4.5 / (6 × (2²³−1))` μV/LSB
- 24bit は「2の補数」なので、最上位ビットが立っていれば負の値として扱う

→ `codec.py` の `_decode_frame()`。ここまでで「時間 × 16ch の μV 波形」ができます。

---

## 2. 窓で切り出す（スライディングウィンドウ）

脳波は刻々と変わるので、**直近の一定区間だけ**を切り出して繰り返し分析します。

- **窓長 (window)** … 一度に分析する長さ。既定 **1秒**(250サンプル)。UIで 0.5〜4秒に変更可
- **スライド幅 (hop)** … 何秒ごとに分析し直すか。既定 **0.25秒**(= 4回/秒更新)。UIで変更可

窓長を長くすると周波数の分解能が上がり滑らかになる一方、反応は遅くなります。短くすると逆。
この「窓を少しずつずらしながら繰り返す」のがスライディングウィンドウです。

---

## 3. 前処理 → FFT → PSD

切り出した窓に対して、チャンネルごとに次を行います。

1. **DC除去(detrend)** … 平均を引いて、電極のオフセット(直流成分)を消す
2. **窓関数(Hann窓)** … 区間の両端をなめらかに 0 へ落とし、切り出しによる偽の高周波を抑える
3. **FFT** … 時間波形を周波数成分に分解する（高速フーリエ変換）
4. **PSD(パワースペクトル密度)** … 各周波数の「強さ」を μV²/Hz で求める（片側スペクトルに正規化）

→ サーバー側は `aggregate.py`(numpy)、ブラウザ側は `fft.ts`(自作の radix-2 FFT)。
両者は同じ手順なので、実機モードとシミュレータモードで近い数値が出ます。

> なぜ scipy.welch でなく単純な periodogram か: デモ規模では十分な精度で、依存を軽くできるため。

---

## 4. バンドパワーに集計する

PSD を、脳波で意味のある **5つの周波数帯**に積分(台形則)してまとめます。

| バンド | 周波数 | ざっくりした意味 |
|---|---|---|
| δ (delta) | 1–4 Hz | 深い睡眠など |
| θ (theta) | 4–8 Hz | まどろみ・記憶 |
| α (alpha) | 8–13 Hz | リラックス・閉眼時に増える |
| β (beta) | 13–30 Hz | 集中・思考・筋電の影響も |
| γ (gamma) | 30–45 Hz | 高次処理・ノイズも乗りやすい |

各バンドについて **ch別の値** と **全ch平均** を出します。棒グラフは全ch平均、頭部マップは ch別を使います。

---

## 5. 集中度・リラックス度

バンドパワーの比から、メンタル状態の指標を計算します（`mind.ts`）。

- **集中度 Focus = β / (α + θ)** … Pope らの engagement index。集中・認知活動で上がる
- **リラックス度 Relax = α / (α + β)** … α 優位度。安静・閉眼で上がる（0〜1）
- **合成 composite** … focus と relax を並べて `-1〜+1` に正規化し、
  `> 0.15` を「集中」、`< -0.15` を「リラックス」、その間を「中立」と判定

値がフレームごとに揺れるので、直近数サンプルの移動平均で滑らかにして表示します。

---

## 6. 空間・時間の可視化

- **頭部トポグラフィ** … ch別バンドパワーを 10-20 電極配置(`montage.ts`)に置き、
  **逆距離加重補間(IDW)**で頭全体を塗り分け。「頭のどこが活発か」を色で見る(`topography.ts`)
- **3D電極マップ** … 同じ ch別パワーを、回転する頭部ドーム上に球の大きさ・色で表示(`brain3d.ts`)
- **認知状態トラジェクトリ(3D)** … focus × relax × time の3D空間に、状態の時間変化を軌跡として描く。
  一点にとどまれば安定、大きく動けば状態がゆらいでいる(`trajectory3d.ts`)

---

## まとめ（一文で）

> 24bitの生波形を μV に直し、直近1秒を Hann窓+FFT で周波数に分解して δ/θ/α/β/γ の強さに集計、
> その比から集中/リラックスを推定し、電極配置に沿って空間・時間で可視化している。
